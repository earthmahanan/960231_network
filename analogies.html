Overview
Real-world analogies for web concepts
Eight everyday scenarios that map to stateless HTTP, cookies, sessions, JWT, localStorage, OAuth 2.0, CDN caching, and browser caching. Expand any card, then jump to its animated walkthrough below.
Analogy 01
Stateless HTTP — The goldfish waiter
Every HTTP request starts from zero. The server has no memory of who you are or what you ordered last time.
Restaurant visit → one request · Waiter forgets you → no persistent state
Analogy 02
Cookies — The theme park wristband
After login the server snaps a wristband on you. Every ride operator reads it automatically — until someone steals it or you cut it off.
Set-Cookie → wristband · Auto-sent Cookie header → show wrist at each ride
Analogy 03
Sessions — The hotel coat check
You hand your coat to the hotel and get ticket #4471. The real data stays in the back room; you only carry the opaque ID.
Session ID cookie → ticket · Redis store → back room · Revoke → hotel refuses ticket
Analogy 04
JWT — Your driver's licence
All claims are printed on the card and signed by the issuer. Anyone can verify the stamp without calling home — but stolen licences work until expiry.
Signed payload → licence · Crypto verify → check stamp · No revocation list
Analogy 05
localStorage — The notebook on your desk
Data persists across sessions but any script on the page can read it. Nothing is mailed to the server unless you explicitly send it.
Survives restart → notebook stays · XSS → stranger reads desk · Never auto-sent
Analogy 06
OAuth 2.0 — The apartment valet key
You never hand your master key to the removalist. The building manager issues a limited valet key you can revoke without changing your password.
Master key → password · Valet key → access token · Scopes → front door only
Analogy 07
Web cache (CDN) — The local convenience store
Popular responses are stocked at the edge node near users. Fast when the shelf is full; stale until TTL expires or the CDN purges.
CDN edge → corner shop · max-age → best-before · Purge → product recall
Analogy 08
Browser cache — Photos saved on your phone
Repeat visits load from disk instead of the network. Hard refresh bypasses the gallery and demands the latest file from the server.
Cache hit → gallery · ETag/max-age → is it recent? · Hard reload → pull-to-refresh
